## Validation Results

### Route-to-Luxon Harness (48 cases)
**E2E Accuracy: 1/48 (2.1%)** — regression from Exp 004's 25%.

#### Failure Modes
1. **System prompt mismatch** (initial run: 0/48): Harness sent `SYSTEM_PROMPT_ROUTE` but model was trained with SmolLM default system prompt. Fixed, reran.
2. **NO_ROUTE over-triggering** (~50% of cases): Model classified most temporal questions as non-computable. The test prompts ("What time is it X after Y?") are formulaic; the training data uses activity-based phrasing. Distribution mismatch.
3. **Numerical errors**: When routes were emitted, argument values were often off (e.g., 17:44 instead of 18:05).
4. **Incomplete routes**: Some outputs missing operands, causing engine parse errors.

## Root Cause Analysis

### 1. Training-Test Distribution Mismatch
The generator produces activity-contextual questions:
> "I stopped gaming at crack of dawn after 30 minutes. When did I start?"

The harness tests formulaic questions:
> "What time is it 1 minute after 23:59?"

The model learned to route domain-specific language but cannot generalise to generic temporal arithmetic questions. These are two non-overlapping distributions.

### 2. Weak NO_ROUTE Discrimination
The NO_ROUTE training data (15%) uses obvious non-temporal questions ("Is 3pm a good time for a meeting?"). The model hasn't learned to distinguish hard cases — temporal questions that *look* computable but aren't (e.g., "When was The Secret Garden published?").

### 3. Catastrophic Forgetting
10K steps of full FT on narrow domain data displaced general reasoning capability without achieving reliable routing in return. The trade-off yielded negative ROI.

## Decisions for Next Experiment (009)

1. **Expand question variety in generator** — add formulaic patterns ("What time is it X after Y?"), edge cases (midnight rollover, minute carry), and diverse phrasing structures
2. **Use published temporal QA datasets as hard negatives** — incorporate ComplexTempQA questions as NO_ROUTE examples (temporal but non-computable)
3. **Consider reducing training steps** to limit catastrophic forgetting; possibly explore earlier checkpoints as sweet spot
4. **Align test harness with training distribution** — either generate harness from generator, or ensure generator covers harness patterns
